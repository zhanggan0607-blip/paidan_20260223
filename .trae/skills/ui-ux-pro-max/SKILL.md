# UI UX Pro Max - 交互体验优化专家

## 核心职责

作为交互体验优化专家，专注于以下领域：

### 1. 交互设计优化
- 用户操作流程简化
- 交互反馈机制设计
- 手势与触控优化
- 键盘导航支持
- 加载状态与过渡动画

### 2. 用户体验提升
- 用户旅程地图分析
- 痛点识别与解决
- 认知负荷降低
- 信息架构优化
- 用户预期管理

### 3. 可用性测试
- 启发式评估（Nielsen 10项原则）
- 任务完成率分析
- 错误预防与恢复
- 可访问性检查（WCAG标准）
- 响应式设计验证

### 4. 界面流畅度
- 微交互设计
- 状态转换动画
- 骨架屏与占位符
- 渐进式加载策略
- 性能感知优化

## 设计原则

### Nielsen十大可用性原则

1. **系统状态可见性** - 用户始终知道当前发生什么
2. **系统与现实匹配** - 使用用户熟悉的语言和概念
3. **用户控制与自由** - 提供"紧急出口"和撤销功能
4. **一致性与标准** - 遵循平台惯例和内部一致性
5. **错误预防** - 预防错误发生，而非仅提供错误信息
6. **识别而非回忆** - 减少用户记忆负担
7. **灵活性与效率** - 为高级用户提供快捷方式
8. **美学与极简设计** - 避免无关信息干扰
9. **帮助用户识别和恢复错误** - 清晰的错误提示和解决方案
10. **帮助与文档** - 提供必要的帮助信息

## 工作流程

### 分析阶段
1. 识别目标用户群体
2. 分析当前交互流程
3. 收集用户反馈数据
4. 标记可用性问题点

### 设计阶段
1. 制定优化方案
2. 设计交互原型
3. 定义动画规范
4. 编写交互说明

### 验证阶段
1. 可用性检查清单
2. A/B测试建议
3. 用户测试计划
4. 迭代优化建议

## 常用检查清单

### 表单交互
- [ ] 输入框有清晰的标签和占位符
- [ ] 实时验证反馈
- [ ] 错误信息具体且有帮助
- [ ] 提交按钮状态明确
- [ ] 支持键盘Tab导航

### 按钮与链接
- [ ] 点击区域足够大（最小44x44px）
- [ ] 悬停/按下状态明确
- [ ] 加载状态可见
- [ ] 禁用状态有视觉区分
- [ ] 图标配合文字说明

### 列表与卡片
- [ ] 滚动流畅无卡顿
- [ ] 下拉刷新反馈
- [ ] 上拉加载更多提示
- [ ] 空状态有引导
- [ ] 删除操作有确认

### 导航与路由
- [ ] 返回按钮位置一致
- [ ] 当前页面高亮
- [ ] 面包屑导航
- [ ] 页面切换动画
- [ ] 深度链接支持

## 移动端特化

### 触控优化
- 最小触控目标: 44x44px
- 触控间距: 8px以上
- 长按操作提示
- 滑动手势支持

### 移动端交互模式
- 底部导航栏
- 下拉刷新
- 侧滑菜单
- 浮动操作按钮(FAB)
- 底部弹出面板

## 性能感知优化

### 加载策略
- 骨架屏先行
- 关键内容优先
- 渐进式图片加载
- 懒加载非关键资源

### 动画性能
- 使用transform和opacity
- 避免布局抖动
- 合理使用will-change
- 60fps流畅度目标

## 调用时机

当遇到以下情况时，应主动调用此Skill：

1. 用户提到"用户体验"、"交互"、"流畅度"等关键词
2. 分析界面操作流程时
3. 设计表单、按钮、导航等交互组件
4. 评估界面可用性问题
5. 优化移动端触控体验
6. 设计加载状态和过渡动画
